I. Introduction to HCI

Human-Computer Interaction (HCI) is a multidisciplinary field focusing on how people interact with computers and digital systems. It studies the design, evaluation, and implementation of interactive technologies.

Key Components:
  • The User: The individual or group involved in a task.
  • The Computer/System: The technology (software, hardware, mobile apps, etc.).
  • The Interaction: The process of the user performing tasks via the system.

HCI Goals & Rationale

Core Goals & Importance

Core Goals:
  • Effectiveness: Does the system do what the user intends?
  • Efficiency: Can tasks be completed quickly with minimal effort?
  • Usability: Is the system easy to learn and remember?
  • Accessibility: Can people with different abilities use the system?
  • Satisfaction: Do users feel positive about using the system?
Why HCI Matters:
  • Reduces Cognitive Load (easier to understand).
  • Minimizes human errors.
  • Ensures technology fits human capabilities rather than forcing humans to adapt to technology.

II. The Evolution (History)

1. Early Era (1940s–1960s): Hardware-Centered

Focus: Machine efficiency and computational power.

  • Batch Processing: Tasks collected and processed all at once; no real-time feedback.
  • Punch Cards: Stiff paper encoding binary data via holes.
  • CLI: Required memorization of exact text commands.
  • Users: Highly trained specialists and engineers only.
2. Birth of Interactive (1960s–1970s)
  • Time-Sharing: Allowed multiple users to share one computer simultaneously.
  • Interactive Terminals: Keyboard/screen for real-time communication.
  • Milestones: Sutherland’s Sketchpad (1963) and Engelbart’s Mouse and Hypertext (1968).

GUI to Modern Era

3. The GUI Revolution (1980s)
  • WIMP Model: Windows, Icons, Menus, Pointer.
  • Visual Metaphors: Desktop, Folders, Trash Bin.
  • Direct Manipulation: Dragging files directly.
4. Usability/Mobile Era (1990s–2010s)
  • Focus shifted from "interfaces" to UX (User Experience).
  • Responsive Design: Adapting layouts for Mobile/Web.
5. Modern HCI: Beyond Screens
  • VUI (Voice), AR (Augmented Reality), VR (Virtual Reality), and BCI (Brain-Computer Interfaces).
  • Shift: Keyboard → Mouse → Touch → Voice → Neural.

III. Principles & Usability

Nielsen’s Five Usability Attributes:

Learnability, Efficiency, Memorability, Errors (recovery), and Satisfaction.

Shneiderman’s 8 Golden Rules:
  • Consistency: Use similar icons/layouts.
  • Shortcuts: For frequent users (Ctrl+C).
  • Informative Feedback: System responses.
  • Dialog Closure: Clear beginning/end.
  • Prevent Errors: Disable Submit if fields empty.
  • Easy Reversal: Undo/Redo.
  • Internal Locus of Control: User feels in charge.
  • Reduce Memory Load: Use autocomplete.

Nielsen’s 10 Heuristics

Nielsen’s 10 Usability Heuristics

  • Visibility of System Status: Loading spinners/status.
  • Match Between System & Real World: Trash Bin metaphors.
  • User Control & Freedom: Emergency exits (Cancel/Undo).
  • Consistency & Standards: Don't reinvent conventions.
  • Error Prevention: Better than good error messages.
  • Recognition Rather Than Recall: Visible options/menus.
  • Flexibility & Efficiency: Accelerators for experts.
  • Aesthetic & Minimalist Design: No clutter.
  • Help Users with Errors: Diagnostic messages.
  • Help & Documentation: Focused on the task.

IV. HCD & User-Centered Process

The UCD Process (Iterative):
  • 1. Understanding Users: Identify goals and limitations.
  • 2. Defining Requirements: Functional/accessibility needs.
  • 3. Design Solutions: Sketches, wireframes, prototypes.
  • 4. Evaluation: Test with real users.
  • 5. Iteration: Improvement based on feedback.
Essential Concepts (Don Norman):
  • Affordance: Actions an object allows (Button affords pressing).
  • Signifier: Cues on how to use it ("Push" label).
  • Constraint: Restricting actions to help avoid mistakes.

V. Cognitive Psychology in HCI

Human Information Processing (HIP):

Input → Perception → Cognition → Memory → Decision → Action → Feedback.

1. Attention:

Selective, limited, disrupted. Types: Focused, Divided, Sustained.

2. Gestalt Principles (Perception):
  • Proximity: Close objects = related.
  • Similarity: Similar looking = grouped.
  • Closure: Users "fill in" missing info.
  • Figure-Ground: Foreground vs Background.

Memory & Decision Making

3. Memory Systems:
  • Sensory: Milliseconds.
  • Short-Term/Working: ~4 chunks capacity.
  • Long-Term: Unlimited knowledge/skills.
  • HCI Rule: Recognition over Recall.
4. Cognitive Load Theory:
  • Intrinsic: Task complexity.
  • Extraneous: Caused by poor design (clutter).
  • Germane: Effort put into learning.
5. Decision & Errors:
  • Hick’s Law: More choices = longer decision time.
  • Slips: Correct goal, wrong action (accidental click).
  • Mistakes: Wrong goal (misunderstanding feature).
